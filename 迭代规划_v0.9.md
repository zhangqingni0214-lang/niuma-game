## 二、当前结局触发规则全解

### 2.1 判定时机

每次 `advanceTime()` 推进（时间槽 +1，或跨日）之后，立即调用 `checkEnding()`。

### 2.2 判定算法

```
1. 取出全部 13 个结局定义
2. 按 priority 倒序排序（数字大的先判）
3. 遍历，第一个 condition(stats, ctx) === true 的结局就是结局
4. 立即调用 finalizeLife() 终止本世
5. 没有命中 → 继续游戏
```

**ctx 上下文**：`{ day, character, job, snarkCount }`。

### 2.3 13 个结局判定条件（优先级倒序）

| Priority | 结局 | 触发条件 |
|---|---|---|
| 100 | 过劳猝死 | fatigue≥100 **且** health≤25 |
| 95 | 拍桌裸辞（小马专属） | character=horse **且** snarkCount≥5 **且** stress≥70 |
| 95 | 老实人之死（小牛专属） | character=ox **且** fatigue≥95 **且** salary≥60 |
| 90 | 精神崩溃裸辞 | stress≥100 **且** mood≤20 |
| 88 | 被自己优化（HR 专属） | job=hr **且** day≥5 **且** salary≤25 |
| 85 | 被开除·但走得有尊严 | salary≤15 **且** mood≥70 |
| 80 | 被优化 | salary≤10 |
| 75 | 副业反超·辞职自雇 | day≥10 **且** money≥5000 |
| 70 | 不符合时代精神 | skill≤15 **且** day≥7 |
| 60 | 破产滚回老家 | money≤−300 |
| 50 | 陷入抑郁 | mood≤10 **且** day≥5 |
| 10 | 活下来了（通关） | day>14 |

### 2.4 优先级设计意图

- **同时多个条件命中时，谁先判到谁赢**。例：玩家同时满足过劳猝死（100）和破产（60），优先过劳。
- **职业/角色专属结局放在 88–95 之间**：保证差异化体验不被通用致死结局（100 的过劳猝死）轻易盖掉。
- **通关 priority=10**：永远兜底，只有所有死亡条件都不触发才落到这里。
- **未来扩展建议**：节日特殊结局（春节回家、年终颁奖）priority 建议 65–75，介于"副业反超"和"被优化"之间。

### 2.5 已知问题

1. **小马拍桌裸辞门槛偏低**：snarkCount≥5 + stress≥70 在 Day 5 前就可能凑齐，导致部分玩家"撒不开手怼"。建议加 day≥5 限制。
2. **副业反超门槛偏高**：money≥5000 在 14 天日薪 + 副业收入下，¥10000+ 月薪的职业才有概率达到。需要观察实际触达率。
3. **小牛"老实人之死"要求 salary≥60**，但小牛的天然 trait 是 salary +1 / fatigue -3，反而拖慢疲劳累积，触发难度偏高。可考虑改为 salary≥50。

下次平衡 review 建议把这 3 条纳入。

## 三、正则误判问题 + Tag 化迁移方案

### 3.1 什么叫"正则误判"

当前 `applyEffects` 里有几条 passive 技能修正用文本正则触发，例如：

```js
// 不眠咒：喝咖啡不扣健康
if (unlocked.has('coffee_immune') && /咖啡|美式|黑咖/.test(choice?.text || '')) {
  if (adj.health && adj.health < 0) adj.health = 0;
}

// 橡皮鸭：技术类事件加技能
if (unlocked.has('rubber_duck') && /加班|凌晨|代码|bug|功能|case/.test(ev?.text + ev?.title || '')) {
  adj.skill = (adj.skill || 0) + 2;
}
```

**问题在于"文本写什么完全是文案的事，但被代码逻辑依赖了"**。举几个真实存在 / 可能存在的误判场景：

| 文案 | 期望行为 | 实际发生 |
|---|---|---|
| "你看了一眼**咖啡**机就转身走开" | 不该触发不眠咒 | ❌ 触发了，因为文本含"咖啡" |
| "PM 说**加班**两天够吧"（其实玩家没加班） | 不该触发橡皮鸭 +skill | ❌ 触发了 |
| "**同事**给你递了根烟" | 群聊魅影 +mood？ | ⚠️ 触发了，但语境无关 |
| "老板**画饼**"（玩家选了"装睡"） | 不该触发反向画饼术 +salary | ❌ 触发了 |

**根本原因**：用文本判定行为是耦合反模式。文案改了，逻辑就跟着错；翻译多语言时整个机制崩盘。

### 3.2 Tag 化迁移方案（v0.9 必做）

#### Step 1：定义事件 / 选项的元数据 tag 集

```js
// 事件级 tag（描述场景类型）
EVENT_TAGS = ['tech', 'meeting', 'overtime', 'commute', 'team', 'client',
              'health', 'leisure', 'side', 'pm', 'boss', 'hr']

// 选项级 tag（描述玩家行为）
CHOICE_TAGS = ['coffee', 'fishing', 'overtime', 'snark', 'submissive',
               'social', 'flatter', 'kpi_grind', 'duck_debug']
```

#### Step 2：事件 / 选项加 tag 字段

```js
{
  id: 'pm_simple_feature',
  title: '这个功能应该很简单吧',
  tags: ['tech', 'pm', 'meeting'],   // 新增
  text: '...',
  choices: [
    {
      text: '【橡皮鸭】当场把需求拆成 7 个子任务表',
      tags: ['duck_debug', 'kpi_grind'],  // 新增
      requiredSkill: 'rubber_duck',
      effects: {...}
    }
  ]
}
```

#### Step 3：把 applyEffects 里的正则改成 tag 检查

```js
// 改前
if (unlocked.has('rubber_duck') && /加班|凌晨|代码|bug|功能|case/.test(ev?.text + ev?.title || '')) {
  adj.skill = (adj.skill || 0) + 2;
}

// 改后
if (unlocked.has('rubber_duck') && ev.tags?.includes('tech')) {
  adj.skill = (adj.skill || 0) + 2;
}
```

#### Step 4：迁移现有 49 条事件

需要给每条事件补 tags 字段。**这一步是手工活，约 1.5 个工时**。建议结合 todo 1（扩到 100 条）一起做：新增事件直接带 tag，老事件随手补。

#### Step 5：清理 trackChoice 里的正则

```js
// 改前
if (/摸鱼|装作|拍照|盯着|画一/.test(text)) {
  h.fishingCount += 1;
}

// 改后
if (choice.tags?.includes('fishing')) {
  h.fishingCount += 1;
}
```

#### Step 6：人设标签条件也走 tag

```js
// 改前
{ id: 'fishing_master', condition: (s, h) => h.fishingCount >= 4 }

// 改后保持不变（fishingCount 已经由 tag 统计而非正则统计）
```

### 3.3 你需要做的

**0 编码工作量**——你只需要 review 我下一步给出的 tag 清单是否合理。我会在补完事件后，一次性发一份 tag 字典 + 全 100 条事件的 tag 标注，你看一眼通过就行。

3. 单独为补覆盖率而硬塞紫色选项，反而容易写得生硬。

## 五、音效方案：拳皇像素 × 颓废感

### 5.1 风格定位

**关键词**：8-bit chiptune × Lo-fi 颓废 × 日式怀旧

**参考对照**：
- 节奏 / 编排骨架：拳皇 '94 / '95 选关 BGM、合金弹头存档间奏
- 情绪基底：《Persona 5》Lo-fi 改编、《赛博朋克 2077》Night City Radio 慵懒频道、《死亡搁浅》Low Roar
- 中文参考：《中国式家长》《灵魂学院》打工模拟器的"打卡音 + 钟声"运用

**调性公式**：拳皇音色（FM 合成器、方波、噪声鼓）+ 拖慢 BPM 到 70–90 + 加入磁带嘶嘶声 / 雨声 / 远处汽车声白噪音 → 颓废感成立。

### 5.2 必做音效清单（v0.9 最小集）

#### A. UI 音效（10 条以内，每条 < 0.5 秒）

| ID | 用途 | 风格说明 |

### 5.3 制作路径（按性价比排序）

**Plan A：付费购买素材包（最快，1–2 天）**
- itch.io 搜 "chiptune sfx" / "lofi 8bit"，付费包 $5–$30
- envato / audiojungle 找拳皇风格 sfx + lofi loop
- 预算：¥200–500
- 优点：立即可用
- 缺点：撞包风险，独特性低

**Plan B：AI 生成 + 二次编辑（折中，3–5 天）**
- Suno / Udio 生成 BGM（提示词："8-bit chiptune, lofi tape hiss, slow 70 BPM, melancholic, KOF inspired"）
- jsfxr / sfxr.me 生成 UI 音效
- Audacity 加磁带嘶声 / reverb 颓废化
- 预算：AI 工具订阅约 ¥100/月
- 优点：可控、独特
- 缺点：需要会用工具

**Plan C：外包独立音乐人（最佳质感，2–4 周）**
- 站酷 / fiverr / 米可音乐 找 chiptune 制作人
- 预算：UI 包 ¥800–1500，BGM 单首 ¥1500–3000
- 优点：质感最好，IP 化潜力
- 缺点：周期长、成本高

**我的建议**：**Plan B + Plan A 混合**——BGM 用 Suno 自己生成 2 首，UI sfx 买现成包微调。总预算 < ¥500，2 周内可完工。

### 5.4 技术集成

```html
<!-- index.html 加 audio 元素 -->

```

```js
// game.js 新增 SFX 模块
const SFX = {
  enabled: true,
  click: new Audio('assets/sfx/click.mp3'),
  snark: new Audio('assets/sfx/snark.mp3'),
  // ...
  play(name) {
    if (!this.enabled || !this[name]) return;
    this[name].currentTime = 0;
    this[name].play().catch(() => {}); // 浏览器自动播放策略
  }
};

// 接入点

showMoneyToast(amount) { ... SFX.play(amount > 0 ? 'toast_money_in' : 'toast_money_out'); }
```
